# Event Handling

The MQL5 language provides handling of certain [predefined events](/en/docs/runtime/event_fire). The functions for handling these events should be defined in an MQL5 program: function name, return type, a set of parameters (if any) and their types should strictly correspond to the description of an event handling function.

The client terminal event handler uses the return and parameter types to identify functions processing an event. If a certain function has some parameters or a return type not corresponding to the descriptions below, such a function cannot be used for handling an event.

| Function | Description |
| --- | --- |
| [OnStart](/en/docs/event_handlers/onstart) | The function is called when the [Start](/en/docs/runtime/event_fire#start) event occurs to perform actions set in the script |
| [OnInit](/en/docs/event_handlers/oninit) | The function is called in indicators and EAs when the [Init](/en/docs/runtime/event_fire#init) event occurs to initialize a launched MQL5 program |
| [OnDeinit](/en/docs/event_handlers/ondeinit) | The function is called in indicators and EAs when the [Deinit](/en/docs/runtime/event_fire#deinit) event occurs to de-initialize a launched MQL5 program |
| [OnTick](/en/docs/event_handlers/ontick) | The function is called in EAs when the [NewTick](/en/docs/runtime/event_fire#newtick) event occurs to handle a new quote |
| [OnCalculate](/en/docs/event_handlers/oncalculate) | The function is called in indicators when the [Calculate](/en/docs/runtime/event_fire#calculate) event occurs to handle price data changes |
| [OnTimer](/en/docs/event_handlers/ontimer) | The function is called in indicators and EAs during the [Timer](/en/docs/runtime/event_fire#timer) periodic event generated by the terminal at fixed time intervals |
| [OnTrade](/en/docs/event_handlers/ontrade) | The function is called in EAs during the [Trade](/en/docs/runtime/event_fire#trade) event generated at the end of a trading operation on a trade server |
| [OnTradeTransaction](/en/docs/event_handlers/ontradetransaction) | The function is called in EAs when the [TradeTransaction](/en/docs/runtime/event_fire#tradetransaction) event occurs to process a trade request execution results |
| [OnBookEvent](/en/docs/event_handlers/onbookevent) | The function is called in EAs when the [BookEvent](/en/docs/runtime/event_fire#bookevent) event occurs to process changes in the market depth |
| [OnChartEvent](/en/docs/event_handlers/onchartevent) | The function is called in indicators and EAs when the [ChartEvent](/en/docs/runtime/event_fire#chartevent) event occurs to process chart changes made by a user or an MQL5 program |
| [OnTester](/en/docs/event_handlers/ontester) | The function is called in EAs when the [Tester](/en/docs/runtime/event_fire#tester) event occurs to perform necessary actions after testing an EA on history data |
| [OnTesterInit](/en/docs/event_handlers/ontesterinit) | The function is called in EAs when the [TesterInit](/en/docs/runtime/event_fire#testerinit) event occurs to perform necessary actions before optimization in the strategy tester |
| [OnTesterDeinit](/en/docs/event_handlers/ontesterdeinit) | The function is called in EAs when the [TesterDeinit](/en/docs/runtime/event_fire#testerdeinit) event occurs after EA optimization in the strategy tester |
| [OnTesterPass](/en/docs/event_handlers/ontesterpass) | The function is called in EAs when the [TesterPass](/en/docs/runtime/event_fire#testerpass) even occurs to handle an arrival of a new data frame during EA optimization in the strategy tester |

The client terminal sends incoming events to corresponding open charts. Also, events may be generated by charts ([chart events](/en/docs/event_handlers/onchartevent)) or mql5 programs ([custom events](/en/docs/eventfunctions/eventchartcustom)). Generating graphical object creation/deletion events can be enabled/disabled by setting the [CHART\_EVENT\_OBJECT\_CREATE](/en/docs/constants/chartconstants/enum_chart_property#enum_chart_property_integer) and [CHART\_EVENT\_OBJECT\_DELETE](/en/docs/constants/chartconstants/enum_chart_property#enum_chart_property_integer) chart properties. Each mql5 application and chart have their own queue of events where all newly arrived events are placed.

A program gets events only from the chart it is running on. All events are handled one after another in the order of their receipt. If the queue already contains the [NewTick](/en/docs/runtime/event_fire#newtick) event or this event is in the processing stage, then the new NewTick event is not added to mql5 application queue. Similarly, if the [ChartEvent](/en/docs/runtime/event_fire#chartevent) is already in an mql5 program queue or such an event is being handled, then a new event of this type is not placed into a queue. Timer event handling is processed in the same way â€“ if the [Timer](/en/docs/runtime/event_fire#timer) event is already in the queue or is being handled, no new timer event is set into a queue.

Event queues have a limited but sufficient size, so the queue overflow is unlikely for a correctly developed program. When the queue overflows, new events are discarded without being set into a queue.

It is strongly recommended not to use infinite loops to handle events. Possible exceptions are scripts handling a single [Start](/en/docs/runtime/event_fire#start) event.

[Libraries](/en/docs#library) do not handle any events.
