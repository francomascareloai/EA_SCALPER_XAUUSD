The [RSIOMA](/en/code/1581) indicator with the timeframe selection option available in input parameters:

```mql5
input ENUM_TIMEFRAMES TimeFrame=PERIOD_H4;  // Indicator chart period (timeframe)
```

The RSIOMA\_HTF repeater indicator requires the compiled custom indicator file RSIOMA.mq5 in order to compile. Place it in *<terminal\_data\_folder>\MQL5\Indicators*.

After compilation, the RSIOMA\_HTF.ex5 indicator file contains the RSIOMA.ex5 indicator as a [resource](/en/docs/runtime/resources), and therefore, it is not required to be present in the terminal folder for the compiled indicator to work! For this purpose, the corresponding code has been added to the indicator code in order to include the RSIOMA indicator in the executable file.

The indicator executable file has been added as resources at the global scope

//---- Include custom indicators in the indicator code as resources  
#resource \\Indicators\\RSIOMA.ex5

Changed the string path to the indicator used as resource in the block of the OnInit() function

//---- getting handle of the RSIOMA indicator  
 Ind\_Handle=iCustom(Symbol(),TimeFrame,"::Indicators\\RSIOMA",  
 RSIOMA\_Method,RSIOMA,RSIOMAPhase,MARSIOMA\_Method,MARSIOMA,MARSIOMAPhase,MomPeriod,IPC,HighLevel,MiddleLevel,LowLevel,0);

Thus, the compiled executable file of the repeater indicator can be used on other trade terminals on its own without the original indicator.
