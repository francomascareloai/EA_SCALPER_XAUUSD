# ADVANCED ELICITATION TASK

## Propósito
Esta task implementa métodos avançados de elicitação para extrair requisitos detalhados e contexto específico do usuário, adaptada para o domínio de trading.

## Cenários de Uso

### 1. Criação de Documentos
Quando o usuário solicita criação de documentos (PRDs, specs técnicas, etc.), use métodos de elicitação para:
- Entender o contexto completo
- Identificar stakeholders
- Mapear requisitos funcionais e não-funcionais
- Descobrir restrições e dependências

### 2. Chat Geral
Em conversas gerais, use elicitação para:
- Clarificar intenções ambíguas
- Descobrir contexto implícito
- Identificar necessidades não expressas
- Validar entendimento

## Métodos de Elicitação

### Seleção Inteligente
Escolha o método baseado no contexto:

1. **Entrevista Estruturada**: Para requisitos complexos
2. **Questionário Direcionado**: Para informações específicas
3. **Análise de Cenários**: Para casos de uso
4. **Prototipagem Conceitual**: Para validação de ideias
5. **Análise de Stakeholders**: Para identificar envolvidos

### Implementação

```yaml
elicitation_methods:
  structured_interview:
    when: "Requisitos complexos ou ambíguos"
    approach: "Perguntas abertas seguidas de específicas"
    
  targeted_questionnaire:
    when: "Informações técnicas específicas"
    approach: "Perguntas fechadas e direcionadas"
    
  scenario_analysis:
    when: "Casos de uso ou workflows"
    approach: "Análise de cenários de trading"
    
  conceptual_prototyping:
    when: "Validação de conceitos"
    approach: "Mockups ou diagramas conceituais"
    
  stakeholder_analysis:
    when: "Identificação de envolvidos"
```

## Loading Strategy

**IMPORTANTE**: ONLY load dependency files when needed

### Quando Carregar
1. Quando agente é ativado pela primeira vez
2. Quando comando específico requer dependency
3. Quando workflow solicita template específico
4. Quando validação requer checklist

### Como Carregar
```yaml
loading_strategy:
  lazy_loading: true
  cache_dependencies: true
  validate_before_load: true
  error_handling: "graceful_degradation"
```
