### /matriz [estrategia] - Extrair Nivel MATRIX

```
O QUE E "NIVEL MATRIX"?
Ã‰ o nivel mais profundo de entendimento de uma estrategia,
onde voce ve a "matrix" por tras da realidade superficial.

WORKFLOW MATRIX:

1. DESCONSTRUCAO TOTAL
   - Quebrar estrategia em componentes atomicos
   - Identificar cada regra e seu proposito
   - Mapear dependencias entre componentes

2. QUESTIONAR CADA PREMISSA
   - Por que essa regra existe?
   - O que acontece se remover?
   - O que acontece se inverter?
   - Existe alternativa melhor?

3. BUSCAR ORIGEM HISTORICA
   - Quem criou originalmente?
   - Em que contexto de mercado?
   - O que mudou desde entao?
   - Ainda faz sentido hoje?

4. TESTAR LIMITES
   - Em que condicoes funciona?
   - Em que condicoes FALHA?
   - Quais os edge cases?
   - Qual o worst case scenario?

5. ENCONTRAR O CORE
   - Qual e a UNICA coisa essencial?
   - O que pode ser removido sem perder o edge?
   - Qual o principio fundamental?

6. RECONSTRUIR OTIMIZADO
   - Manter apenas o essencial
   - Adicionar melhorias baseadas em pesquisa
   - Simplificar onde possivel
   - Documentar cada decisao

OUTPUT: Estrategia no nivel mais fundamental possivel

## 3.3 Conhecimento de ML para Trading

```
FRAMEWORKS E FERRAMENTAS:

TREINO/DESENVOLVIMENTO:
- Python + PyTorch/TensorFlow
- Pandas/NumPy para dados
- Scikit-learn para features
- Optuna para hyperparameters
- MLflow para tracking

VALIDACAO:
- Walk-Forward Analysis (WFA)
- Purged K-Fold Cross-Validation
- Combinatorial Purged CV
- Monte Carlo simulation

DEPLOYMENT:
- ONNX para exportar
- MQL5 para inferencia
- Latencia < 5ms obrigatorio

ARQUITETURAS QUE FUNCIONAM:
- LSTM (classico, funciona)
- GRU (mais rapido que LSTM)
- Transformer (novo, promissor)
- XGBoost (baseline forte)
- Ensemble (combinar varios)

RED FLAGS EM ML TRADING:
- Accuracy > 80% (overfitting)
- Sem OOS testing
- Sem custos de transacao
- Apenas 1 ativo/periodo
- Features futuras (look-ahead)
- Muito parametros (>10)
```

## 3.4 Conhecimento de Backtesting

```
METODOLOGIAS DE VALIDACAO:

1. WALK-FORWARD ANALYSIS (WFA)
   - Padrao ouro da industria
   - IS: 70% / OOS: 30%
   - 10-20 janelas minimo
   - WFE >= 0.6 para aprovar

2. PURGED K-FOLD CV
   - Evita data leakage
   - Gap entre folds
   - Melhor que K-Fold simples

3. MONTE CARLO
   - 5000+ simulacoes
   - Shuffle de trades
   - 95th percentile DD

4. ROBUSTEZ DE PARAMETROS
   - Variar parametros Â±20%
   - Resultado deve ser estavel
   - Se quebra com variacao = overfit

ERROS COMUNS:
- Look-ahead bias (usar dados futuros)
- Survivorship bias (so ativos vivos)
- Selection bias (periodo favoravel)
- Overfitting (muitos parametros)
- Ignorar custos reais
```
